The Vision

While some Flight Simulator fans are interested in using the moving map display on a GPS device to record and track the progress of a simulated flight, fsr represents my interest in using the playback feature of Flight Simulator to review and analyze actual GPS tracks of flight maneuvers and approaches I make while practicing my private pilot skills - to "replay" my flights in Flight Simulator. By the way - that's me (in the picture) in Skyward Aviation's AA1-C trainer, during my first solo landing in front of the tower at KSMO!

Now, almost every time I fly, I take my GPS handheld device in the plane with me and leave it in "track record" mode. When I come back home, I can "replay" my flight (or portions of it) in Flight Simulator by using fsr. I can use Flight Simulator's alternate views (e.g., cockpit, spot plane, map. etc.) to get different perspectives about my flight, and I can watch my ground track, groundspeed, altitude and other aspects from the comfort of my armchair as my actual experience is simulated in the replay.

The Reality

fsr is actually the result of a desire to embrace The Vision, and to learn to program in the Microsoft .NET framework, in C#. That is why you need Microsoft's .NET Framework in order to run fsr (fsr was developed on .NET Framework version 1.0 but should work with version 1.1 as well). Currently, fsr works only with Microsoft Flight Simulator 2002 and Microsoft Flight Simulator 2004.

While I've reached many of the goals I set out to achieve when I first started creating fsr, the product is currently less useful than I had hoped (and still hope) it could be someday. Technical difficulties that take time to diagnose and circumvent (mainly in interfacing with Flight Simulator) slowed development, and many such problems are still at large. The two areas I would most like to make improvements are:

Increased Playback Smoothness
The simulated airplane flys a very choppy/jittery path using both supported methods for replaying the flight in the simulator. I'm confident that the data I'm sending to the simulator - a default of 18 frames per second - are smooth (as I've demonstrated through statistical analysis), so I'm suspecting the simulator needs some other data that I'm not currently providing, to make it happy.

Increased Realism
Because the GPS only records position, altitude, groundspeed and course, I'll never be able to see or analyze some of the details of the actual experience that I'd like to - such as airspeed, bank angle, use of the controls or the engine settings, etc. However, for aesthetic reasons, I've tried to identify some of the easiest of these things to infer mathematically. For instance, fsr currently imputes pitch and bank, but this is at best just "eye candy" - the actual realism of the replay isn't improved since these are not based upon actual observations.

The Future

In addition to solving the remaining problems, the following enhancements are currently envisioned:
Multiplayer Playback Method
In "multiplayer playback mode", fsr would create a "player" that would appear in a Flight Simulator Multiplayer Session, flying the recorded track. In this mode, other players in the session could interact with (or observe) the fsr-generated flight from different perspectives than are now possible using the other two play methods.

Integrating Other Real-time Data Sources
Richer and/or external data streams could be synchronized with the GPS track data collected from the flight and merged into the playback. For example:
  • The data stream sent to the flight data recorder on commercial aircraft (such as actual pitch/bank angles and control inputs) could provide detail that could be used to produce a very realistic simulation of an actual flight.
  • Audio or other time-sequenced materials (such as ATC interactions) related to the actual flight could be synchronized with the simulator playback in order to increase the "next best thing to being there" effect. One idea was to enable the pilot to create "timestamps" by pressing a button on the GPS device at key moments during an actual flight. Later, these timestamps could be annotated and these annotations displayed within the simulator at the appropriate time within the flight replay.

Support for other Simulators
Current candidates are Flight Gear and X-Plane, but the concept easily extends itself to terrestrial and marine simulators as well.
